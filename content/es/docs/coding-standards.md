# Estándares de Código

Este documento describe los estándares de código y mejores prácticas para el equipo Yu-Dev.

## Principios Generales

- Escribir código limpio y legible
- Seguir las convenciones específicas del lenguaje
- Comentar la lógica compleja
- Mantener las funciones pequeñas y enfocadas

## Revisión de Código

Todos los cambios de código deben:

- Pasar las pruebas automatizadas
- Ser revisados por al menos un miembro del equipo
- Seguir la guía de estilo establecida
- Incluir documentación apropiada

## Control de Versiones

- Usar mensajes de commit significativos
- Mantener los commits atómicos y enfocados
- Crear ramas de características para nuevo trabajo
- Mantener la rama principal estable
